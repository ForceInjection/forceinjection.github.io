# Agent Skills (AI 技能)

**Agent Skill**（智能体技能）是指封装了特定领域知识、最佳实践、自动化工作流及多角色协同机制的可复用认知模块。在现代 AI 辅助开发中，单靠基础的 Prompt 往往难以应对复杂的工程挑战（如架构逆向工程、深度代码理解、规范驱动开发等）。通过将这些复杂任务拆解并固化为一个个独立的 Agent Skill，大模型可以像调用 API 一样加载这些技能，从而极大地提升分析深度、准确性与工作效率。

本模块汇总了我们在实际工程中沉淀和验证过的核心技能集合，包含领域驱动设计（DDD）、CUDA 编程辅助以及各类工程提效的实用技能。这些资源能够帮助开发者更好地利用大模型构建复杂的应用与智能系统。

## 核心技能库概览

### 1. Awesome Skills

一个由原力注入博主精心精心整理的通用 AI Agent 技能集合，涵盖了日常研发流中的高频痛点。

**链接**: [本地文档](./awesome-skills/README.html)，[GitHub 仓库](https://github.com/ForceInjection/awesome-skills)

**技能列表**:

| 技能名称                                           | 核心用途                        | 工作流与特点                                                                       |
| -------------------------------------------------- | ------------------------------- | ---------------------------------------------------------------------------------- |
| **深度代码阅读** (`code-reader`)                   | 系统性阅读与理解陌生代码库      | 引入三重协作，以“闭卷考试”机制深入解析模块，避免浅尝辄止。                         |
| **项目架构分析** (`project-analyzer`)              | 第三方代码库静态分析与逆向工程  | 生成包含系统架构、核心模块解析、执行流及基础设施配置的客观深度分析报告。           |
| **目录结构整理** (`dir-organizer`)                 | 规范化项目目录与文件重构        | 基于严格状态收集与计划确认，自动更新移动文件后的内部引用链接。                     |
| **文档自动化审查** (`doc-reviewer`)                | 审查技术文档准确性与专业性      | 包含大纲、内容、资产、格式四种独立评审类型，并支持自动应用修复建议。               |
| **Markdown 总结器** (`md-summarizer`)              | 本地文档分析与结构化提取        | 对单文件进行核心概要、深度解析提取，支持多文件对比分析与主题提炼。                 |
| **智能代码提交助手** (`update-submitter`)          | 生成 Conventional Commits       | 分析 `git diff` 自动逻辑分组变更，获取授权后自动执行标准化的 Git 提交。            |
| **Agent Skill 审查器** (`agent-skill-reviewer`)    | 规范化 Agent Skill 的编写与结构 | 自动审查技能的目录命名、文件结构、指令清晰度，并输出结构化改进报告。               |
| **OpenSpec 规范开发辅助** (`openspec-assistant`)   | 规范驱动的架构设计与验证        | 支持架构师、开发和 QA 的多角色协同，覆盖意图对齐、代码实现与自动化验证。           |
| **网页内容下载器** (`web-content-downloader`)      | 将网页转存为离线 Markdown       | 保留原网页语言与结构，自动提取和存储网页图片，适合构建离线知识库。                 |
| **Markdown 翻译器** (`md-translator`)              | 技术文档的专业翻译              | 专门针对 Markdown 格式设计的本地文档多语言翻译器，自动处理语言标识后缀。           |
| **参考文献整理助手** (`reference-organizer`)       | 生成标准化参考文献列表          | 自动获取链接（含 arXiv、DOI 等）元数据，并根据博客、白皮书、论文要求生成引用。     |
| **Markdown 链接检查器** (`md-link-checker`)        | 文档链接的健康度检测            | 批量检查本地与外部链接的可访问性，保障技术文档的长期可维护性。                     |
| **Draw.io 架构图设计** (`drawio-designer`)         | 自动化系统架构图与流程图绘制    | 自动集成 AWS、K8s 等 IT 架构图标，创建并编辑 Draw.io XML 图表并转换为 PNG。        |
| **PPTX 读取器** (`pptx-reader`)                    | 演示文稿内容分析                | 直接解析 `.pptx` 幻灯片文件内容，提取文本或辅助进行演示文稿分析。                  |
| **知识图谱本体管理** (`ontology`)                  | 维护基于本体的图谱记忆结构      | 支持创建与查询实体、关联依赖，为跨技能调用和复杂规划提供长期结构化记忆（Memory）。 |
| **杂志编辑信息卡设计** (`editorial-card-designer`) | 响应式高密度数据卡片渲染        | 设计现代杂志风格的 HTML 信息卡片，并自动渲染输出多种固定视觉比例的 PNG 封面图。    |
| **技术文章大纲规划** (`tech-outline-planner`)      | 内容结构与行文逻辑规划          | 结合 Context-first 和过程叙事结构，为系统设计或技术文章自动生成高质量的大纲。      |

### 2. Domain-Driven Design (DDD) Skills

聚焦于领域驱动设计（DDD）架构思想落地的专属技能组。

**链接**: [本地文档](./domain-driven-design-skills/README.html)，[GitHub 仓库](https://github.com/ForceInjection/domain-driven-design-skills)

**能力说明**:

| 技能名称 / 能力域    | 核心用途       | 核心价值                                                       |
| -------------------- | -------------- | -------------------------------------------------------------- |
| **统一语言提取**     | 领域语言规范化 | 协助架构师从业务需求和对话中提取一致的 Ubiquitous Language。   |
| **上下文边界划分**   | 系统拆分与隔离 | 自动规划 Bounded Context，理清模块间的依赖与通信方式。         |
| **聚合根与实体建模** | 领域模型设计   | 帮助开发者识别核心实体与值对象，构建高内聚的 Aggregates。      |
| **模型代码映射审查** | 落地一致性保障 | 在复杂代码重构中，持续审查领域模型与代码库实现之间的双向映射。 |

**链接**: [本地文档](./cuda-code-skill/README.html)，[GitHub 仓库](https://github.com/ForceInjection/cuda-code-skill)

**能力说明**:

| 技能名称 / 能力域   | 核心用途             | 核心价值                                                                        |
| ------------------- | -------------------- | ------------------------------------------------------------------------------- |
